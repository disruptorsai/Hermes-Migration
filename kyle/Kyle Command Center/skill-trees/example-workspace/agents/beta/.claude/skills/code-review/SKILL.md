# Code Review

When reviewing code:

1. Check for correctness — does it do what it claims?
2. Check for security — any injection, auth, or data exposure issues?
3. Check for readability — would someone else understand this in 6 months?
4. Check for edge cases — what inputs could break this?
5. Be specific — point to exact lines, suggest alternatives
